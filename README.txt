public - можно вызывать как извне смарт-контракта, путём отправки специальной транхакции, так и изнутри контракта, т.е. другая функция может вызывать данную функцию
external - можно обращаться только извне, но изнутри обратиться нельзя
internal -  можно обращаться только изнутри, но не извне, а также к internal функции могут обращаться те контракты, которые наследуют данный контракт
private - только изнутри контракта, но не из его потомков

view - данная ф-ия может только читать данные из блокчейна, но не модифицировать их (считать баланс на смарт контракте, считать значение какой-то переменной и вернуть пользователю), вызывается через call (не нужно платить)
pure - ф-ия так же вызывает как call, но не может читать никакие внешние данные

    // payable - может принимать деньги (тело не обязано содержать логику, средства будут автоматически перечислены на баланс контракта)
    // transaction (используются для модификации каких-либо данных, перевода/принятия денежных средств, берут газ за вызов)
    //call (чтение данных, произведение вычислений, не связанных с внешними данными, но не берут газ за вызов)
